# hamster-skin-designer 子智能体模板

这些文件是项目级子智能体提示模板。主 agent 看到复杂任务时，按 `AGENTS.md` 的自动分工规则选择对应模板派发。

## 默认组合

- `architecture.md`：确认需求边界、文档约束、责任层和影响面。
- `implementation.md`：在明确写入范围内做最小补丁。
- `verification.md`：按回归矩阵选择验证命令并报告结果。
- `review.md`：审查 diff、风险、漏测和项目规则违规。

## 使用规则

- 简单任务不拆分；复杂任务开始前，主 agent 必须说明是否拆分。
- 以下情况默认拆分：跨责任层、preview/export/import 同源链路、Jsonnet、schema、默认值、导出语义、UI 预览验证、测试失败、回归审查。
- 子智能体输出只作为证据和建议；最终取舍由主 agent 负责。
- 多个子智能体并行时，写入范围必须互不重叠。
- 不允许子智能体回滚他人改动。

## 闭环要求

- 架构 agent 先给责任层、影响面和禁止改动边界。
- 实现 agent 只改主 agent 指定文件范围。
- 验证 agent 跑对应命令；未跑必须写原因。
- 审查 agent 基于最终 diff 找高置信问题。
- final 汇总必须写明：派发了哪些 agent、各自结论、采纳/未采纳原因、验证结果、剩余风险。
