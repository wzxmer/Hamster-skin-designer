# 竖屏中文 26 键 iOS 基准预览重做设计

## 背景

当前工作台左侧模块对右侧预览的属性控制已经较完整，但右侧生成出来的预览图观感明显不足。问题不在于“有没有渲染出结构”，而在于默认效果不像真实手机输入法皮肤：候选区、工具栏、键盘、气泡、上下划动状态的整体质感弱，导致预览图和实际应用效果都不够可信。

本轮用户明确要求：

- 不要再只重构框架、但视觉效果没有变化；
- 默认布局也要重新设计；
- 参考 `https://ryanwuson.github.io/rime-liur-ios-skin/` 的预览完整度；
- 第一刀先做竖屏中文 26 键；
- 视觉方向选择“真机 iOS 基准”，密度选择“A2 平衡默认”；
- 工具栏图标要采用 Apple SF Symbols 风格，可参考当前 26 键示例；
- 暗黑模式、气泡、上下划动功能都要纳入 26 键预览闭环，但按阶段一个个来。

## 目标

第一轮只围绕竖屏中文 26 键，做出肉眼可见的默认预览效果提升。

成功的默认预览应满足：

- 候选区、工具栏、键盘看起来是一个完整输入法面板，而不是独立控件拼接；
- 默认布局比例接近 iOS 系统键盘，使用 A2 平衡密度；
- 字母键、功能键、enter 键、底排比例、行缩进、按键阴影和底边缘都重新设计；
- 工具栏使用 SF Symbols 风格图标，不使用 emoji 或普通文字图标代替；
- 浅色、暗黑、按下气泡、长按气泡、上下划动字符与划动气泡最终都能在 26 键预览中看到；
- 修改默认数据和预览渲染，而不是只改一层 CSS。

## 非目标

- 不在第一轮处理九键、数字键盘、符号键盘、Emoji、横屏。
- 不做整套 `preview.js` 大重构。
- 不照搬参考项目的页面结构或运行时依赖。
- 不让参考站成为本项目依赖；本项目仍必须能作为 GitHub Pages 纯静态网页运行。

## 设计方向

### 视觉方向

采用“真机 iOS 基准”：

- 默认键盘应像用户日常见到的 iOS 输入法；
- 参考项目用于补足完整面板、状态层级和预览完整度；
- 不追求参考项目的装饰性风格。

### 默认密度

采用“A2 平衡默认”：

- 候选区约 42px；
- 工具栏约 38px；
- 字母键约 40px；
- 行间距约 6px；
- 左右内边距约 5px；
- 底部内边距略大于顶部，保留真实输入法的托底感。

这些数值是设计基准，实现时应映射到 `project.sample.json` 的 `keyboardFrame`、`buttonInsets`、`surfaceStyles`、`metrics` 与预览 CSS/渲染逻辑中。

## 阶段设计

### 阶段 1：默认 26 键面板

范围：

- 竖屏中文 26 键；
- 候选区、工具栏、键盘三段整体面板；
- 浅色模式；
- 默认行布局和底排比例。

效果要求：

- 面板背景使用 iOS 风格浅灰；
- 字母键为白色实体键；
- 功能键为灰色实体键；
- enter 键为 iOS 蓝；
- 按键有轻微底边缘、边线和阴影；
- 中排有真实缩进；
- 下排 shift/backspace 宽度更接近真机；
- 底排空格占主视觉，功能键比例稳定。

### 阶段 2：工具栏 SF Symbols

范围：

- 只处理 26 键预览里的 toolbar；
- 优先复用 `renderSystemImageSvg` 和现有 `SYSTEM_IMAGE_BY_KEY` 映射；
- 默认 toolbar 图标改成 SF Symbols 风格。

效果要求：

- 图标线重、尺寸、垂直居中统一；
- 不再用 emoji 或普通文字模拟图标；
- `menu`、`symbol`、`translate`、`emoji`、`phrase`、`pasteboard`、`script`、`close` 都应有清晰图标；
- toolbar 背景融入完整面板，不像一排孤立按钮。

### 阶段 3：暗黑模式

范围：

- 竖屏中文 26 键；
- 候选区、工具栏、键盘；
- 与浅色使用同一布局比例。

效果要求：

- 面板背景为深灰；
- 字母键为半透明深灰/中灰实体键；
- 功能键层级略深或略浅，和字母键可区分；
- 文字为白色或高亮灰白；
- enter 键继续保持蓝色；
- 阴影和底边缘在暗色下不能脏。

### 阶段 4：气泡

范围：

- 按下气泡；
- 长按气泡；
- 26 键默认字符键与功能相关键。

效果要求：

- 气泡有真实浮层感：圆角、阴影、边缘、和按键之间的空间关系清楚；
- 浅色/暗黑分别有合理背景和文字；
- 长按选中项有 iOS 蓝色高亮；
- 气泡文字位置和大小应匹配当前按键字号，不挤、不漂。

### 阶段 5：上下划动功能

范围：

- 26 键按键上的上划/下划字符；
- 划动气泡预览；
- 与现有 swipe/hint 行为数据保持兼容。

效果要求：

- 上划/下划字符在按键内有固定位置和弱化颜色；
- 字母主字符和划动字符层级分明；
- 划动气泡能显示当前方向触发的字符或符号；
- 不改变现有上下划动作语义，只改善默认数据和预览呈现。

## 数据与代码边界

必须同步考虑：

- `packages/project-schema/defaults/project.sample.json`
- `apps/workbench/src/ui/preview.js`
- `apps/workbench/styles.css`
- 与 SF Symbols 相关的 `packages/preview-engine/index.js` 或已有调用点

第一轮可以改默认数据、渲染函数和样式，但不得引入后端、本地文件依赖或参考站运行时依赖。

## 验收方式

每个阶段都应完成：

- 当前阶段视觉截图；
- 与改动前的默认 26 键预览肉眼对比；
- 浅色/暗黑阶段需要分别验收；
- 运行基础验证命令。

建议验证命令：

```powershell
rtk npm run validate:project-schema
rtk npm run test:preview
rtk npm run build
```

如果改到导出默认数据或 exporter 相关逻辑，还要运行：

```powershell
rtk npm run test:exporter
```

## 成功标准

- 竖屏中文 26 键默认预览第一眼接近真实 iOS 输入法，而不是“网页里拼出的键盘”；
- 默认布局、默认颜色、默认阴影、默认图标、默认状态都有肉眼变化；
- 工具栏、暗黑模式、气泡、上下划动在 26 键范围内形成完整闭环；
- 后续九键、数字、符号、横屏可以沿用这一套视觉标尺继续推进。
