# 项目记忆与文档存放规则

## 触发口令

- `记住这个：...`：整理当前信息并写入项目记忆
- `搜索记忆：关键词`：检索已有项目记忆
- `恢复记忆`：恢复当前项目最相关的上下文
- `更新记忆`：补充本阶段新增的结论、决策和约束
- `整理文档` / `同步一下`：同步项目文档、约定与记忆

## 存放规则

1. 项目记忆与相关文档存放在使用者自己的本地环境中。
2. 不要把个人电脑绝对路径写入公开仓库。

## 当前项目落点

当前项目的本地记忆落点只在私有环境中维护，公开仓库只保留通用规则。

## 执行约定

- 需要沉淀长期上下文时，优先写入对应项目的 `memory` 目录，并同步到可检索记忆体系
- 需要保留说明、约束、阶段结论时，优先写入对应项目的 `docs` 目录
- 如果用户没有额外指定路径，默认按本规则自动归档
